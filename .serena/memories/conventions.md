# Code conventions

- **Svelte 5 runes only**: `$state`, `$derived`, `$props()` — no legacy `export let` / stores-as-writable pattern for component props. The one app-wide mutable store (`appState`) is a `$state` object exported from `app-state.svelte.ts`; components read it via `appState.x` and mutate only through the exported functions, not inline, except for simple UI toggles like `appState.tab = ...`.
- **Design tokens**: all colors/radii/gradients come from `TOKENS` in `src/lib/tokens.ts` (imported per-component) or CSS custom properties (`var(--color-*)`, `var(--radius-*)`, `var(--layout-*)`) defined globally. Never hardcode hex colors in a `<style>` block if a token exists.
- **Real vs mock data fallback pattern** (see `getPlayerRows` in `review.ts` for the canonical example): prefer real parsed/computed data, fall back to the `mock-data.ts` fixture only when the real value is null/unavailable. Always comment *why* the fallback exists and that it's a fallback, not the primary source.
- **`GameData.isSample` gate**: any feature that depends on data the real engine/parser doesn't produce yet (classification, coach text, breakdown/phase tables) must only render "real-looking" mock content when `game.isSample` is true; a genuinely different pasted game gets real positions/moves but explicit "not available yet" messaging (`UNCLASSIFIED_COACH_TEXT` pattern) rather than misleading mock numbers. Follow this same gate for any new real-data wiring.
- **Testing**: co-located `ComponentName.test.ts` next to `ComponentName.svelte`; pure-logic modules under `src/lib/game`/`src/lib/charts`/`src/lib/board` also get `*.test.ts` siblings. TDD is expected — tests describe behavior before/alongside implementation, not retrofitted.
- **Rust**: doc comments on structs/functions reference the design-handoff doc sections that justify them (e.g. `OVERVIEW §6.5`, `LOGIC.md §7`) — keep that traceability when adding new Rust code.
- **Commit style**: Conventional Commits (`feat:`, `fix:`, `refactor:`, `docs:`, `chore:`) — see `git log` for house style; no AI co-author trailer (user strips it).
